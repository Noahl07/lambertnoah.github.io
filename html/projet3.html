Projet Pépi+ – Application Web de Gestion de Stock

Le projet Pépi+ consiste à concevoir une application web complète permettant la gestion des stocks réels et virtuels d’une pépinière, avec traçabilité sanitaire obligatoire. Le projet a été réalisé sous Symfony (architecture MVC) avec une base MySQL.

Contexte métier

La société Pépi+ produit et revend des plants (arbres, arbustes, vivaces). Chaque plant doit être tracé par :

  • Son essence (nom latin + nom commun)
  • Sa saison de production
  • Son conditionnement (pot normalisé ou racine nue)
  • Sa provenance (production interne ou partenaire fournisseur)

L’objectif est d’assurer une gestion fiable du stock et de garantir la traçabilité sanitaire, exigence réglementaire du secteur.

Architecture technique

  • Framework : Symfony (MVC)
  • SGBD : MySQL
  • Architecture : 3-tiers (Client / Serveur applicatif / Base de données)
  • Authentification : Système sécurisé par rôles (Admin / Collaborateur / Partenaire)

Le projet respecte les bonnes pratiques : séparation contrôleurs/services, validation des formulaires, gestion des accès, et organisation claire des entités.

Modélisation (Conception BDD)

La base de données a été conçue via un MCD permettant d’assurer la traçabilité complète. Les principales entités :

  • Plant (essence, saison, conditionnement)
  • Stock réel
  • Stock virtuel (partenaires)
  • Commande
  • Ligne de commande
  • Partenaire
  • Utilisateur (rôles)

La relation entre LigneCommande → Plant → Saison → Partenaire permet de prouver la traçabilité exigée dans le cahier des charges :contentReference[oaicite:1]{index=1}.

Fonctionnalités développées

  • Gestion du référentiel des plants (CRUD complet)
  • Gestion des saisons
  • Gestion des partenaires
  • Visualisation globale du stock (réel + virtuel)
  • Réservation automatique lors d’une commande
  • Mise à jour dynamique des quantités
  • Modification / suppression d’une commande avec impact sur le stock
  • Tableau de bord synthétique pour le pilotage

Points techniques avancés

  • Contrôles d’accès via rôles Symfony
  • Validation serveur + gestion erreurs métier
  • Requêtes optimisées pour calculer les stocks en temps réel
  • Respect strict du pattern MVC
  • Jeu de données de test complet (≥10 enregistrements par table)
  • Gestion Git rigoureuse (1 commit par fonctionnalité)
