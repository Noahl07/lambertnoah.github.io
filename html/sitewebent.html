Projet — Site web Tecnitub (Stage)

Développement d’une application web sous Symfony 7 (MVC) avec espace client, interface administrateur, sécurité applicative avancée et déploiement OVH (Linux).

Entreprise : Tecnitub
Période : 5 janvier 2026 – 6 février 2026 (6 semaines)
Contexte : Entreprise sans service informatique interne (réalisation en autonomie)
Stack : Symfony 7, PHP, Twig, MySQL (Doctrine), HTML/CSS, JavaScript
Production : OVH mutualisé (Linux), configuration DocumentRoot /public

Présentation de l’entreprise

Tecnitub est une entreprise spécialisée dans la chaudronnerie, la tuyauterie industrielle, le chauffage et la climatisation. Elle intervient principalement auprès de professionnels et d’industries nécessitant des installations techniques sur mesure.

L’entreprise ne disposant pas de service informatique, le projet a été développé en autonomie, avec validation et retours du maître de stage.

Objectifs techniques

  • Finaliser une architecture MVC Symfony maintenable.
  • Mettre en place un espace client sécurisé (authentification, demandes).
  • Développer une interface administrateur (gestion demandes / fichiers).
  • Renforcer la cybersécurité (CSRF, rate limiting, uploads sécurisés).
  • Déployer en production OVH et stabiliser le fonctionnement.

Architecture & données

Application Symfony structurée (Controllers / Entities / Templates Twig). Base de données MySQL gérée via Doctrine ORM.

  • Entités : User, demandes/commandes, Contact, ResetPasswordRequest.
  • Relations : User (1) → (N) demandes/commandes.
  • Intégrité : contraintes d’unicité email, validations, cohérence BDD ↔ fichiers uploadés.

Fonctionnalités (principales)

  • Authentification : inscription, vérification email, login/logout sécurisés.
  • Compte client : consultation de ses demandes, téléchargement de fichiers associés.
  • Demandes : création / suivi / demande de modification.
  • Admin : gestion des demandes, édition, actions (selon implémentation), gestion fichiers Excel.

Sécurité applicative

La sécurité a été traitée comme une contrainte centrale : réduction des surfaces d’attaque (auth, formulaires, uploads) + protections côté serveur.

  • CSRF sur formulaires (y compris logout), validation serveur.
  • Rate limiting (anti-abus) + protections contre l’énumération d’emails.
  • Mots de passe : hash sécurisé + règles de robustesse.
  • Uploads : limite taille/nombre, contrôle MIME, whitelist extensions, scan ZIP, renommage aléatoire, stockage hors public.
  • Accès : rôles (ROLE_USER / ROLE_ADMIN) et contrôles d’accès.

Déploiement (OVH) & mise en production

Mise en production sur OVH mutualisé (Linux) avec gestion des environnements Symfony.

  • Configuration DocumentRoot → /public + vérification réécriture (htaccess).
  • Variables d’environnement (.env), désactivation debug, nettoyage cache.
  • Gestion permissions serveur + validation du chargement des assets (CSS/JS).

Incidents rencontrés & corrections

  • Erreur 404 sur route admin (ex. /admin/clients) : contrôle des routes disponibles + vérification de l’exposition réelle des URL + ajustements navigation/accès.
  • Production : erreurs de configuration (variables / chemins / cache) → diagnostic + corrections.
  • Linux : sensibilité à la casse (fichiers/images) → standardisation des noms.

Ces corrections démontrent une capacité à diagnostiquer (routes, configuration, logs) et stabiliser une appli en production.

Erreur 404 – Diagnostic en production

Capture d’écran d’un incident rencontré lors d’un test d’accès à une route administrateur non exposée (/admin/clients). Cette situation a nécessité une analyse des routes Symfony, des contrôles d’accès (rôles) et de la configuration en production.

Espace client – Gestion des demandes

Interface utilisateur permettant la consultation des demandes, téléchargement des fichiers associés et demande de modification.

Interface administrateur – Gestion globale

Vue administrateur centralisant l’ensemble des demandes clients : gestion des statuts, actions de modification, suppression/restauration et accès aux fichiers Excel associés.

Modification d’une commande

Écran de modification permettant l’édition des informations, la mise à jour du statut (en attente / en cours / terminé) ainsi que la gestion du fichier Excel associé.

Bilan technique

Le projet a abouti à un site web professionnel déployé en production, intégrant une architecture MVC Symfony, un espace client et une interface administrateur. La mise en production a permis de confronter le projet à des contraintes réelles (Linux, configuration, sécurité).

Compétences renforcées : Symfony (structure projet, sécurité), gestion des données (Doctrine/MySQL), sécurisation (CSRF, uploads), diagnostic production (routes, configuration, incident).
